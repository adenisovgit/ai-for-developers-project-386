# Проект «Запись на звонок» (Call Booking)

Это монорепозиторий для упрощенного сервиса бронирования времени по мотивам Cal.com. 

## Бизнес-логика
- Один владелец: публикует доступное время (слоты по 30 минут), смотрит предстоящие встречи.
- Гость (пользователь): видит свободные слоты, выбирает время, оставляет контакты, записывается на звонок.
- **Ограничения:** Нет системы авторизации, нет личных кабинетов, нет внешних интеграций с Google Calendar и прочим. Максимально простое, но законченное решение.

## Технологический стек и Архитектура
1. **Frontend (`/frontend`)**: Vue 3 (Composition API / `<script setup>`), Vite.
2. **Backend (`/backend`)**: Java + Spring Boot (последняя стабильная версия, инициированная через Spring Initializr), Gradle. In-memory база данных (H2).
3. **API (`/api`)**: Проектируется через **TypeSpec**. Источник правды — `openapi.yaml`.
4. **Контейнеризация**: Проект собирается и работает в Docker (через `docker-compose.yml` в корне).

## Правила для AI-Агентов

### 1. Архитектура и Код
1. **API-First Approach**: API является первичным. Сначала меняется TypeSpec, собирается OpenAPI.
2. **Backend**: Spring-контроллеры и DTO должны **генерироваться** из OpenAPI (использовать плагин `openapi-generator-gradle-plugin`). Агент НЕ ДОЛЖЕН писать контроллеры вручную, только реализацию делегатов/сервисов.
3. **Frontend**: HTTP-клиент и типы TypeScript должны **генерироваться** из OpenAPI.
4. Выполняйте код с учетом работы в средах JetBrains (IntelliJ IDEA Ultimate, WebStorm) и локальных AI-утилит.
