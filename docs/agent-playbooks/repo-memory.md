# Repo Memory

Используйте этот playbook для любой нетривиальной задачи в репозитории.

## Workflow

1. Перед существенными изменениями прочитайте `PROJECT_MEMORY.md`.
2. Сверьте запрос с текущими правилами и зафиксированными решениями проекта.
3. Если код и память проекта расходятся, определите, что должно быть обновлено в рамках задачи: код, документация или оба слоя.
4. После выполнения задачи обновите `PROJECT_MEMORY.md`, если изменилось устойчивое правило или важное состояние проекта.

## Когда нужно обновлять PROJECT_MEMORY.md

- изменения API-контракта
- изменения источника правды или цепочки генерации
- изменения frontend flow
- изменения инфраструктурных или tooling-ожиданий
- изменения ключевых бизнес-правил
- изменения важных UX-договоренностей

## Что не нужно складывать в PROJECT_MEMORY.md

- временные заметки для дебага
- детали одноразовых багов
- общие правила работы агента, не привязанные к этому репозиторию
