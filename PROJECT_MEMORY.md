# Состояние проекта (Project Memory)

Документ для отслеживания архитектурных решений, состояния API и важных изменений в проекте.

## Текущее состояние
- Инициализирован монорепозиторий.
- Созданы базовые правила для AI в `AI_INSTRUCTIONS.md`.
- Создана папка `api/` с инициализацией TypeSpec (`main.tsp`).
- Модули Backend и Frontend пока не инициализированы (находятся в `DEVELOPMENT_PLAN.md` на шагах 3 и 4).

## Важные архитектурные решения и договоренности
- **API-First**: Источником правды является схема TypeSpec.
- **Кодогенерация**: Spring-контроллеры, DTO на бэкенде и HTTP-клиент с типами на фронтенде должны строго генерироваться из `openapi.yaml`.
- **Разделение процесса**: Агенты должны делать короткий план перед выполнением, учитывать имеющиеся наработки (через чтение памяти проекта) и обязательно обновлять этот документ по мере внедрения новых инфраструктурных, архитектурных или бизнес-ограничений.
