## Шаг 1. Инициализация монорепозитория и подготовка AI-артефактов [ЗАВЕРШЕН]
**Цель:** Подготовить проект к работе с AI-агентами (Claude, Codex, Antigravity) в IDE (IntelliJ Ultimate, WebStorm) и настроить базовую структуру директорий.
**Результаты:** 
- Создан файл контекста `AI_INSTRUCTIONS.md`.
- Создана папка `api/` с инициализированным проектом TypeSpec (`package.json`, `main.tsp`).

## Шаг 2. Проектирование API (TypeSpec)
**Цель:** Описать контракт (API-First подход) для взаимодействия компонентов.
**Задачи для выполнения в чате:**
1. Описать доменные модели (Слот, Встреча/Бронирование).
2. Спроектировать REST-эндпоинты:
   - `GET /slots` — получить список доступных временных окон.
   - `POST /bookings` — забронировать время.
   - `GET /meetings` — посмотреть список предстоящих встреч (сценарий владельца).
3. Настроить компиляцию TypeSpec в спецификацию `openapi.yaml`.

## Шаг 3. Backend (Spring Boot + in-memory DB)
**Цель:** Создать серверную часть.
**Стек:** Java, Spring Boot (последняя стабильная сборка на момент генерации), Gradle, H2 Database.
**Задачи для выполнения в чате:**
1. Сгенерировать базовый проект через Spring Initializr в папке `backend/`.
2. Настроить `openapi-generator-gradle-plugin` так, чтобы контроллеры и DTO генерировались автоматически из `api/openapi.yaml`.
3. Реализовать логику хранения бронирований и генерации слотов.
4. Настроить CORS для локальной разработки.

## Шаг 4. Frontend (Vue 3)
**Цель:** Создать пользовательский интерфейс.
**Стек:** Vue 3 (Composition API / `<script setup>`), Vite.
**Задачи для выполнения в чате:**
1. Инициализировать Vite+Vue приложение в папке `frontend/`.
2. Сгенерировать HTTP-клиент и TypeScript-типы из `api/openapi.yaml`.
3. Разработать экраны:
   - Публичная страница пользователя: календарь/список дней, выбор слота (30 мин), форма контактов.
   - Страница владельца: просмотр запланированных созвонов.
4. Уделить приоритетное внимание визуальному качеству (современный дизайн, анимации наведения, красивые цвета).

## Шаг 5. Контейнеризация (Docker)
**Цель:** Подготовить проект к развертыванию.
**Задачи для выполнения в чате:**
1. Добавить `Dockerfile` для `backend/`.
2. Добавить `Dockerfile` для `frontend/` (multistage сборка с Nginx).
3. Создать в корне репозитория скрипт `docker-compose.yml` для одновременного локального старта.
