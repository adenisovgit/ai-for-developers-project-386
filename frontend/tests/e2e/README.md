# Playwright E2E

Интеграционные сценарии для `Шага 5` живут здесь и запускаются через `@playwright/test`.

## Обязательные сценарии

- `happy path` бронирования через публичную страницу и проверка результата в `/admin`
- конфликт слота с серверным `409`, повторной загрузкой занятости и сбросом устаревшего выбора

## Технические договоренности

- frontend для e2e поднимается через `vite preview` на `http://127.0.0.1:4173`
- backend для e2e поднимается на `http://127.0.0.1:8080`
- frontend получает `VITE_API_URL=http://127.0.0.1:8080`
- весь e2e-контур работает в `UTC`
- в CI используется `workers: 1`
- reset endpoint не используется
- изоляция сценариев строится на уникальных датах и уникальных `guestEmail`

## MCP workflow

- `Playwright MCP` используется для ручного прогона сценариев, подбора устойчивых локаторов, проверки фокуса и воспроизведения конфликтов
- `Chrome Devtools MCP` используется для анализа консоли, сети, CORS и повторных запросов после `409`
- автопроверки в CI выполняются только через `@playwright/test`
